# Investally Brand Style Guide

## Overview
This style guide is based on the Investally logo, which symbolizes growth, protection, and partnership in wealth creation. The design elements combine upward-trending arrows, protective hands, and ascending bar charts to represent financial progress and security.

## üéØ Logo Usage

### Logo Variants

#### Full Logo
- Use for headers, hero sections, and primary branding
- Minimum width: 180px
- Clear space: Equal to height of the logo on all sides

#### Logo with Tagline
```
INVESTALLY
YOUR PARTNER IN WEALTH CREATION
```
- Use for footers and formal documents
- Tagline should be 25% of main logo height

#### Logo Icon Only
- Use for favicons, app icons, social media avatars
- Minimum size: 32x32px
- Keep the upward arrow and bar chart elements

### Logo Colors

#### Primary (Colored)
- Use on white or light backgrounds
- Teal (#14B8A6) for "INVEST" and graphic elements
- Black (#0f172a) for "ALLY"
- Green (#84cc16) for bar charts

#### Reverse (White)
- Use on dark backgrounds (Slate 900 or darker)
- Maintain contrast ratio of at least 4.5:1

#### Monochrome
- Use when color is not available
- Single color: Teal 700 (#0d9488)

### Don'ts
- ‚ùå Don't stretch or distort the logo
- ‚ùå Don't change the colors
- ‚ùå Don't add effects (shadows, outlines, etc.)
- ‚ùå Don't place on busy backgrounds
- ‚ùå Don't use low-resolution versions

## üé® Accessibility

### Contrast Ratios
- **Normal text**: Minimum 4.5:1
- **Large text** (18pt+): Minimum 3:1
- **Interactive elements**: Minimum 3:1

### Focus States
```css
/* All interactive elements */
focus: {
  outline: 2px solid #14b8a6;
  outline-offset: 2px;
}
```

### Color Blindness Considerations
- Never use color alone to convey information
- Use icons, patterns, or text labels alongside colors
- Test with color blindness simulators

## üìã Best Practices

### Do's ‚úÖ
- Use the teal color family as the primary brand color
- Maintain consistent spacing using the 4px grid
- Use rounded corners for a friendly, approachable feel
- Implement smooth hover animations (300ms)
- Keep typography hierarchy clear and consistent
- Use white space generously
- Ensure all interactive elements have clear hover states

### Don'ts ‚ùå
- Don't use pure black (#000000) - use Slate 900 instead
- Don't mix border radius styles within the same component
- Don't use more than 3 font weights on a single page
- Don't ignore hover states on interactive elements
- Don't use busy background patterns
- Don't make buttons smaller than 44x44px (touch target)

## üì± Mobile-First Considerations

### Typography Adjustments
```css
/* Mobile (< 768px) */
h1: 2.5rem;   /* Reduce from 3.75rem */
h2: 2rem;     /* Reduce from 3rem */
body: 1rem;   /* Keep standard */

/* Tablet (768px - 1024px) */
h1: 3rem;
h2: 2.5rem;

/* Desktop (> 1024px) */
/* Use full scale */
```

### Spacing Adjustments
- Mobile: Reduce padding by 25%
- Tablet: Reduce padding by 15%
- Desktop: Full spacing

## üéØ Performance Guidelines

### Image Optimization
- Use WebP format with JPG fallback
- Lazy load images below the fold
- Provide multiple sizes for responsive images
- Compress images to < 200KB when possible

### Font Loading
- Use `font-display: swap` for custom fonts
- Subset fonts to only required characters
- Preload critical fonts

### Animation Performance
- Use `transform` and `opacity` for animations
- Avoid animating `width`, `height`, `top`, `left`
- Use `will-change` sparingly
